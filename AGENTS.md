# budgetApp AGENTS

## 프로젝트 설명
- 이 저장소는 CSV 기반 가계부 CLI 앱 프로젝트다.
- 입력 데이터는 `data/*.csv` 파일이며, 거래 추가, 합계 계산, 카테고리 필터링, CSV 로드 같은 기능을 다룬다.

## 코딩 규칙
- 모든 함수와 공개 메서드에는 타입 힌트를 반드시 붙인다.
- 함수 하나의 길이는 최대 50줄을 넘기지 않는다.

## TDD 규칙
- 구현 전에 반드시 테스트를 먼저 작성한다.
- 새 기능은 `pytest`로 실패하는 테스트를 만든 뒤 구현을 시작한다.

## 품질 규칙
- 순환복잡도는 10 이하로 유지한다.
- 복잡도가 높아지면 함수 분리 또는 로직 단순화를 먼저 검토한다.

## 품질 검수 규칙
- 커밋 전에 반드시 `qa_engineer` 서브에이전트로 품질 검수를 수행한다.
- 검수에서는 테스트 누락, 타입 힌트 누락, 복잡도 초과, 엣지 케이스를 확인한다.

## 테스트 실행 명령어
- `pytest`
- `radon cc`

## 커밋 규칙
- 하나의 기능이 개발되면 즉시 커밋하고 푸시한다.
- 기능 단위로 커밋을 나누고, 하나의 커밋에 여러 기능을 섞지 않는다.
